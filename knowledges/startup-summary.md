# 项目启动摘要 — LLM Translator

> 立项基线文档。本文档由 `prodflow-project-start` 在立项阶段产出，记录项目定位、第一版范围与非目标，供后续迭代引用。

- **ISSUE / 版本**：立项阶段（无 issue）
- **日期**：2026-06-30
- **技术形态**：Chrome 浏览器扩展（Manifest V3 + WXT + Vue 3 + TypeScript）

## 1. 项目目标

构建一个基于 LLM 的 Chrome 浏览器翻译插件，允许用户配置指定的云端大模型接口与本地大模型接口，通过 LLM 提供高质量翻译，覆盖全文翻译、划词翻译、小窗输入翻译等场景。

## 2. 目标用户

兼顾两类用户：

- **普通阅读用户**：开箱即用，期望 AI 翻译质量优于传统机翻，默认提供云模型配置即可使用。
- **技术用户 / 开发者**：自带 API Key，可接入本地模型（如 Ollama），注重隐私可控与 Prompt 可深度自定义。

第一版需在「易用性」与「可配置性」之间取得平衡：默认体验对普通用户友好，高级配置对技术用户开放。

## 3. 核心业务链（第一版闭环）

```
用户阅读外文页面
  → 划词选中文本
  → 选区附近出现浮动小触发按钮（"译"）
  → 用户点击触发按钮
  → 调用用户配置的 LLM 接口（云端或本地）
  → 浮层展示译文
```

**交互模型**：划词后不自动翻译，而是显示一个小触发按钮，由用户点击触发翻译，避免打扰阅读流。

**目标语言**：默认采用浏览器当前首选语言（`navigator.language`），用户可在设置中覆盖。

支撑链路：LLM 提供方配置（新增 / 编辑 / 启用 / 测试连通性）。**早期不内置任何模型接口**，仅支持用户配置自定义 LLM 接口。

## 4. 第一版成功标准

1. 划词翻译可用：划词后出现触发按钮，点击后返回译文并浮层展示。
2. LLM 配置可用：支持至少一种云端 OpenAI 兼容接口与一种本地接口（Ollama），可切换并测试连通性。不内置任何模型接口。
3. 目标语言：默认使用浏览器首选语言，可在设置中覆盖。
4. 隐私可控：用户 Key 仅存本地，默认不外传第三方。
5. 可加载运行：能以「加载已解压扩展」方式在 Chrome 中运行。

## 5. 非目标（第一版明确不做）

- 全文翻译（涉及整页 DOM 改造，复杂度高，延后至下一版）。
- 小窗输入翻译（独立交互入口，延后至下一版）。
- 浏览器扩展商店上架与付费分发。
- 跨浏览器（Firefox / Edge）兼容性适配（WXT 框架预留能力，但第一版只在 Chrome 验证）。
- 用户账号体系与云端同步。

> 注（2026-07-06 更新）：上述为第一版（v0.1）非目标，本身不变；随产品演进部分已重规划——「商店上架」转为 v0.3 in-scope（免费分发作为增长渠道），「付费分发」仍为非目标；「跨浏览器」纳入 v0.3 一并打通；「用户账号体系与云端同步」进一步延后至免费产品成熟、变现渐进引入时再建（非"永不做"）。详见 [product-wiki/roadmap/index.md](product-wiki/roadmap/index.md)、[product-wiki/business/PRICING-PLAN.md](product-wiki/business/PRICING-PLAN.md)。

## 6. 关键风险

| 风险 | 说明 | 应对 |
|------|------|------|
| MV3 Service Worker 生命周期 | SW 会被回收，长连接 / 状态需持久化 | 翻译调用走短期请求，状态用 `chrome.storage` 持久化 |
| 跨域请求 | 调用第三方 LLM 接口需声明 host_permissions | 在 manifest 声明用户配置的接口域名（动态权限） |
| 本地模型连通性 | Ollama 默认端口跨域与可达性 | 文档说明本地启动方式，配置项支持自定义 BaseURL |
| Key 安全 | API Key 泄露风险 | 仅存 `chrome.storage.local`，不写入日志、不上传 |
| 翻译质量不稳定 | LLM 输出可能与原文混杂 | Prompt 工程约束只输出译文，提供重译 |

## 7. 技术形态决策

- **Manifest V3**：MV2 已被 Chrome 废弃，强制采用 MV3。
- **WXT 框架**：提供热重载、跨浏览器构建、TS 友好、约定式目录，降低 MV3 脚手架成本。
- **Vue 3 + TypeScript**：符合默认前端栈，组件化开发选项页 / 弹窗 / 浮层。
- **Monorepo 预留**：当前为单一扩展包，但目录结构按可演进组织（`packages/`、`apps/` 预留），便于后续抽离共享逻辑（如 LLM 适配层、翻译核心）。

## 8. 默认决策记录

- **为何选择 MV3 + WXT**：MV3 是 Chrome 现行标准且强制方向；WXT 相比原生手写脚手架可显著降低 content-script / background / popup 的组织成本，并保留跨浏览器能力。
- **为何第一版只做划词翻译**：划词翻译是最高频、最小可验证闭环；全文翻译涉及整页 DOM 重写与样式隔离，风险与收益在第一版不匹配；小窗输入翻译是独立交互，可后续增量。
- **延后事项**：全文翻译、小窗输入翻译、商店上架、跨浏览器验证、账号体系。
- **未执行外部操作**：未创建远程仓库、未发布商店、未申请任何云模型 API Key、未部署任何服务；待用户授权后再进行。
