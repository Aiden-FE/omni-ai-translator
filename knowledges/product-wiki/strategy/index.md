# 产品策略 — LLM Translator

## 愿景

让任何人都能用上最好的大模型翻译，且完全掌控模型来源与数据去向。

## 价值主张

| 用户 | 痛点 | 价值 |
|------|------|------|
| 普通阅读用户 | 传统机翻生硬、不懂上下文 | LLM 语义级翻译，译文更自然 |
| 技术用户 | 现有翻译插件模型固定、不可控 | 自带 Key、接本地模型、Prompt 可定制 |

## 关键能力

- LLM 提供方配置（云端 OpenAI 兼容 + 本地 Ollama）——**不内置任何模型接口**，仅支持用户配置自定义接口
- 划词翻译：划词后出现浮动触发按钮，点击翻译
- 目标语言默认浏览器首选语言，可覆盖
- Prompt 自定义

## 增长

- 第一版开源免费，翻译核心与免 Key 兜底做满，作为增长引擎与口碑来源。
- 增长依赖口碑与技术社区传播（自带 Key 用户群体）。客户端开源是增长资产。
- 商业化延后收费，但方向已锁定（见「商业模式」），路线图预留变现接缝。

## 防御性

- 模型无关的适配层设计，新模型快速接入。
- 隐私可控定位形成用户信任壁垒。
- **变现护城河上移到后端**：客户端开源可 fork，但 fork 复制不了云同步后端与社区内容（Prompt 市场）。越 fork 越扩散 Free 层。

## 商业模式

**Freemium + 订阅，现在定策略、延后收费**。第一版开源免费聚焦验证翻译体验；商业化方向已明确——BYO Key 架构下不在「翻译调用」本身定价，变现点落在「调用之外的后端同步与体系价值」（多设备云同步、术语表、翻译记忆、Prompt 市场、优先路由兜底），类比 Obsidian 客户端免费 + Sync 付费。Pro 随 v0.3 历史+云同步+账号体系就绪后开放，约 $4/月。

完整方案见 [business/PRICING-PLAN.md](../business/PRICING-PLAN.md)。
