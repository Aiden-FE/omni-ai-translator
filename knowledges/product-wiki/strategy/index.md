# 产品策略 — LLM Translator

## 愿景

让任何人都能用上最好的大模型翻译，且完全掌控模型来源与数据去向。

## 价值主张

| 用户 | 痛点 | 价值 |
|------|------|------|
| 普通阅读用户 | 传统机翻生硬、不懂上下文 | LLM 语义级翻译，译文更自然 |
| 技术用户 | 现有翻译插件模型固定、不可控 | 自带 Key、接本地模型、Prompt 可定制 |

## 关键能力

- LLM 提供方配置（云端 OpenAI 兼容 + 本地 Ollama）+ 内置免 Key 免费翻译源（Google/微软，v0.2 起，全新安装默认微软）
- 划词翻译：划词后出现浮动触发按钮，点击翻译
- 目标语言默认浏览器首选语言，可覆盖
- Prompt 自定义

## 增长

- 第一版开源免费，翻译核心与免 Key 免费源做满，作为增长引擎与口碑来源。
- 增长依赖口碑与技术社区传播（自带 Key 用户群体）与商店分发。客户端开源是增长资产。
- **前期纯免费聚焦产品美观可用**：中小竞品密集的市场里，早期任何变现压力（含软付费墙 / Pro 预告）都会驱赶潜在客群；先靠发布速度与免费产品力抢占市场，变现后置。
- **产品美观是增长抓手**：v0.3 上线前全面美化应用，引入 shadcn/ui + tailwindcss 重构 UI，弃用灰白黑冷色调、改适度明亮鲜艳（不抢眼）的主题——在中小竞品密集的市场以视觉吸引力突围。
- 商业化方向保留但**渐进式叠加**，不预设 Pro 基建时间点（见「商业模式」）。

## 防御性

- 模型无关的适配层设计，新模型快速接入。
- 隐私可控定位形成用户信任壁垒。
- **近期护城河**：适配层 + 隐私可控 + 产品美观可用 + 发布速度（自动化商店发布流水线支撑高速迭代）。
- **后端护城河延后**：客户端开源可 fork，但 fork 复制不了云同步后端与社区内容（Prompt 市场）——该护城河随未来变现渐进引入时再建，不在近期排期。

## 商业模式

**渐进式叠加增值付费**。前期翻译核心全免费，聚焦增长与产品美观可用；增值付费功能（多设备云同步、术语表、翻译记忆、Prompt 市场、优先路由兜底等）随产品成熟与用户基数扩大**渐进式引入**，不预设 Pro 基建时间点。BYO Key 架构下不在「翻译调用」本身定价，变现点落在「调用之外的后端同步与体系价值」，类比 Obsidian 客户端免费 + Sync 付费。
