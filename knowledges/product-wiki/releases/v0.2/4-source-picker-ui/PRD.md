## 1. 摘要

本 PRD 定义配置页的「翻译源选择 UI」：把 v0.1 仅面向 LLM 的配置页扩展为支持多类源（LLM、传统 API Key、免 Key 兜底）的统一选择与管理界面，包含源类型选择、当前生效源展示、连通性测试扩展。让用户在一个界面看懂「我现在用的是什么源、能切换到什么源、各源是否可用」。

## 3. 背景

v0.1 配置页（`entrypoints/options/App.vue`）只面向 LLM 提供方：标题「LLM 提供方」、提示「本插件不内置任何模型接口」、源类型下拉只有 openai-compatible / ollama、字段为 name/baseUrl/model/apiKey。这套 UI 无法承载 v0.2 新增的传统源与免 Key 兜底。

v0.2 引入三类源后，配置页必须升级：
- 用户要能选择源类型（LLM / 传统 API Key），并理解免 Key 兜底是「未配置时的默认行为」。
- 用户要看清「当前生效的是哪个源」——是自配的 LLM、传统 API Key，还是兜底。
- 连通性测试要覆盖新增源类型。

本 PRD 是用户可感知的前端入口，依赖功能事项 1（适配层与类型扩展）、2（兜底源）、3（传统 API Key 配置）的底层数据与能力。本 PRD 不引入新底层能力，只做配置页交互整合。

## 4. 目标

**目标**：配置页支持源类型选择、当前生效源展示、连通性测试覆盖新增源类型，让用户清晰管理多类翻译源。

**对齐愿景**：产品策略强调「易用性与可配置性平衡」——配置页是可配置性的门面，要让普通用户看得懂（兜底开箱即用）、让技术用户配得深（LLM/Key/端点）。

**关键结果（SMART）**：

| 编号 | 关键结果 | 衡量方式 |
|---|---|---|
| KR1 | 配置页源类型下拉包含 LLM（openai-compatible/ollama）与传统（google/microsoft） | UI 审查 |
| KR2 | 当前生效源清晰可见，区分「自配源」与「兜底源」 | UI 审查 + e2e |
| KR3 | 连通性测试对 LLM、传统 API Key、兜底源均生效 | 手动测试 |
| KR4 | 兜底状态有明确提示（如「未配置自有源，正在使用免 Key 兜底」） | UI 审查 |

## 5. 细分市场

- **普通阅读用户**：打开配置页应能一眼看懂「现在用的是兜底，想更好就配个源」。本 PRD 降低理解成本。
- **技术用户 / 开发者**：配置页要支持他们精细配置 LLM/Key/端点，不被兜底逻辑干扰。

**约束**：
- 配置页是 Vue 3 SFC（`options/App.vue`），保持既有交互模式（卡片式 provider 列表）。
- 不破坏 v0.1 既有配置流程。

## 6. 价值主张

| 客户任务 / 需求 | 客户获得 | 客户避免的痛点 | 我们做得更好的地方 |
|---|---|---|---|
| 看懂当前用哪个源 | 当前生效源一目了然 | 不知翻译走的是 LLM 还是兜底 | 显式区分自配源与兜底源 |
| 切换源 | 同页选择类型、填配置、启用 | 多处分散配置 | 单页统一管理所有源类型 |
| 知道源是否可用 | 连通性测试覆盖所有类型 | 配了源却不知能否连通 | 测试按钮对每类源生效 |

## 7. 解决方案

### 7.1 UX / 原型

**整体结构**：沿用 v0.1 配置页分区（默认目标语言 + 源管理），将「LLM 提供方」分区升级为「翻译源管理」。

**关键交互**：
- 源类型下拉：openai-compatible / ollama / google / microsoft。
- 选 LLM 类型：字段 name/baseUrl/model/apiKey（沿用 v0.1）。
- 选传统类型（google/microsoft）：字段 name/apiKey/baseUrl（baseUrl 默认官方端点，可改；apiKey 可留空走免 Key 兜底）。
- 当前生效源：在页面顶部或源卡片上显式标注「当前生效」；若生效的是兜底源，标注「免 Key 兜底（未配置自有源）」。
- 连通性测试：每个源卡片「测试连通」按钮，结果 inline 展示。
- 启用/删除：沿用 v0.1。

> 详细 UX 设计（差异性交互、状态、反馈、可访问性、视觉原型）待 issue 创建后由 prodflow-prd revise 模式补全（参考 prodflow-ux-evaluate 技能产出）。

### 7.2 核心功能

**7.2.1 源类型选择**
- 扩展源类型下拉项，覆盖 LLM 与传统源。
- 根据所选类型动态显示对应字段（LLM 显示 model，传统源不显示 model）。

**7.2.2 当前生效源展示**
- 读取 `settings` 当前生效源标识，在 UI 高亮。
- 当无自配源生效、走兜底时，展示「免 Key 兜底」状态提示，并引导用户配置自有源以提升体验。

**7.2.3 连通性测试扩展**
- 「测试连通」对传统源（含免 Key 与有 Key）生效，复用 `test-provider` 通道。
- 测试结果区分成功（译文）/失败（错误提示）。

**7.2.4 提示文案更新**
- 移除「本插件不内置任何模型接口」的旧提示，改为说明兜底与可配置源的关系（如「未配置时使用免 Key 兜底翻译；配置自有源后覆盖兜底」）。

### 7.3 技术

- 改造 `entrypoints/options/App.vue`：源类型下拉扩展、条件字段渲染、生效源展示、提示文案。
- 复用 `getProviders/setProviders/getSettings/setSettings`（`shared/storage.ts`）。
- 连通性测试复用 `chrome.runtime.sendMessage({type:'test-provider'})`。

### 7.4 假设

| 假设 | 验证方式 | 失败信号 |
|---|---|---|
| 卡片式列表可承载新增源类型 | UI 审查 | 字段差异致布局混乱 |
| 用户能理解「兜底」概念 | UX 评估 + 用户反馈 | 用户困惑当前生效源 |

> 视觉规范沿用 [knowledges/ux/design-system.md](../../../../ux/design-system.md)（浮层/卡片/输入框/按钮/响应式），本 PRD 仅记录增量差异。

### 交互逻辑（差异）

通用配置页提供方管理（增删改、启用、测试连通）引用 [knowledges/ux/interaction-patterns.md](../../../../ux/interaction-patterns.md)，不重复展开。本 PRD 差异：

1. **分区标题更名**：「LLM 提供方」→「翻译源管理」，提示文案由「本插件不内置任何模型接口…」改为「未配置时使用免 Key 兜底翻译；配置自有源后覆盖兜底」。
2. **源类型下拉扩展**：openai-compatible / ollama / google / microsoft 四项。
3. **条件字段渲染**：选 LLM 类型显示 name/baseUrl/model/apiKey；选传统类型显示 name/baseUrl/apiKey（不显示 model），apiKey placeholder 明示「留空使用免 Key 兜底；填入则走官方 API」。
4. **当前生效源显式展示**：在「翻译源管理」分区顶部增加「当前生效源」提示行——展示当前生效源名称；若生效的是兜底源，展示「免 Key 兜底（文本将外传到 Google/微软）」并附隐私提示与「配置自有源」引导。
5. **连通性测试扩展**：每个源卡片「测试连通」按钮，结果 inline 展示（✅ 译文 / ❌ 错误），覆盖 LLM、传统 API Key、兜底源。

### 业务约束

- 当前生效源解析：优先用户启用的自有源；无自有源时为免 Key 兜底。
- 兜底源不可编辑（无卡片），仅在生效源提示行展示状态。
- 启用任一自有源后，生效源提示行切换为该自有源，兜底提示消失。
- 隐私透明（设计原则 4）：生效源为兜底时必须明示文本外传去向。

### 状态变化（差异）

| 状态 | 反馈 |
|---|---|
| 无自有源（兜底生效） | 顶部提示「当前生效：免 Key 兜底（文本外传到 Google/微软）」+ 引导配置自有源 |
| 已启用自有源 | 顶部提示「当前生效：<源名称>」；该源卡片启用按钮高亮 |
| 测试中 | 「测试中…」（沿用 v0.1） |
| 测试成功 | ✅ <译文>（沿用 v0.1） |
| 测试失败 | ❌ <错误>（沿用 v0.1，错误经第 1 项统一错误模型归一化） |

### 设备适配（差异）

无差异。配置页最大 720px 居中，沿用 [knowledges/ux/design-system.md](../../../../ux/design-system.md)。顶部生效源提示行在窄屏下自动换行。

### 可访问性（差异）

- 生效源提示行用文字 + 语义标签（非纯颜色）表达兜底/自有源状态，符合 [knowledges/ux/accessibility.md](../../../../ux/accessibility.md)。
- 源类型下拉变更后，条件字段需保持可 Tab 聚焦顺序合理。
- apiKey 输入掩码 + placeholder 说明留空语义。
- 隐私提示文本对比度满足 WCAG AA。

### 验收补充

- 源类型下拉含 4 项，按类型条件渲染字段。
- 顶部生效源提示行可见，兜底状态明示外传去向 + 引导。
- 连通性测试覆盖 LLM、传统 API Key、兜底源。
- 旧提示「不内置任何模型接口」已移除，替换为兜底关系说明。
- v0.1 既有 LLM 配置可见可编辑（回归）。
