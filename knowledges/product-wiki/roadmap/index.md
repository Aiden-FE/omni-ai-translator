# 路线图 — LLM Translator

## v0.1 — 划词翻译 MVP（第一版，立项范围）

- [x] MV3 + WXT + Vue 3 + TS 骨架
- [x] LLM 提供方配置（云端 OpenAI 兼容 + 本地 Ollama，不内置模型）
- [x] 划词翻译：浮动触发按钮 + 点击翻译 + 浮层展示
- [x] 目标语言默认浏览器首选语言，可覆盖
- [x] 连通性测试

## v0.2 — 翻译源配置闭环（进行中）

- [ ] 统一翻译源适配层（LLM + 传统翻译源 provider 抽象与路由）
- [ ] 免 Key 免费翻译源（内置 Google + 微软免 Key 公共端点，默认 microsoft，开箱即用，用户可切换）
- [ ] 传统 API Key 配置（Google/微软填自有 Key，作为可选生效源）
- [ ] 配置页翻译源选择 UI（源类型选择、生效源展示、连通性测试扩展）

> 本轮调整 v0.1「不内置任何模型接口」原则：内置传统翻译免 Key 公共端点作为用户可选的免费翻译源，无隐式自动回退。隐私声明相应更新。

## v0.3 — 历史记录 + 多设备云同步 + 账号体系(Pro 前置基建)

> 由变现方案 WTP 优先级驱动重排(2026-07-01 采纳):此为最高 WTP 项,且为 Pro 收费前置基础设施。原 v0.3「小窗输入翻译」顺延至 v0.4,原 v0.4「全文翻译」顺延至 v0.5。

- [ ] 翻译历史记录(本地 + 可序列化,为云同步预留)
- [ ] 账号体系(注册 / 登录,Pro 许可校验基础)
- [ ] 多设备云同步(历史、设置、术语表、Prompt)
- [ ] 数据结构可序列化 / 可增量同步改造(变现接缝约束)
- [ ] **隐私声明更新**(账号、同步数据、支付通道)——上线前必须调用 `privacy-policy` 技能更新

## 变现接缝（2026-07-01 新增）

> 来源：[business/PRICING-PLAN.md](../business/PRICING-PLAN.md)。核心原则——**现在定策略,不收费;但路线图须为未来 Pro 预留接缝。**

**变现接缝约束（v0.2-v0.4 功能设计须遵守）**：

- 历史记录、设置、Prompt 等数据结构须**可序列化、可增量同步**,不锁死纯本地,为 V1 接入云同步后端预留。
- 不在「翻译调用」上设计量/付费墙;翻译核心与免 Key 免费源保持 Free 全功能。
- Free 层对 Pro 能力用「软上限 + 样本展示」(如 Prompt 市场只读、术语表限额),不设硬付费墙。

**WTP 驱动的路线图重排（2026-07-01 已采纳）**：

按支付意愿,真正驱动付费的是「历史记录 + 多设备云同步 + 账号体系」(WTP 排序第 1),而原 v0.3「小窗输入」、原 v0.4「全文翻译」WTP 偏弱。**已把「历史 + 云同步 + 账号体系」提到 v0.3**,小窗输入顺延至 v0.4、全文翻译顺延至 v0.5——它既是最高 WTP 项,又是 Pro 收费的前置基础设施。同步后端成本在 v0.3 规划时由开发评估。

**Pro 上线触发点**：v0.3 历史+云同步+账号体系就绪后开放 Pro 预售/早鸟。v0.3 上线前须同步更新隐私声明(账号、同步数据、支付通道)。
