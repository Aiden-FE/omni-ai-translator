# 路线图 — LLM Translator

## v0.1 — 划词翻译 MVP（第一版，立项范围）

- [x] MV3 + WXT + Vue 3 + TS 骨架
- [x] LLM 提供方配置（云端 OpenAI 兼容 + 本地 Ollama，不内置模型）
- [x] 划词翻译：浮动触发按钮 + 点击翻译 + 浮层展示
- [x] 目标语言默认浏览器首选语言，可覆盖
- [x] 连通性测试

## v0.2 — 翻译源配置闭环（已完成）

- [x] 统一翻译源适配层（LLM + 传统翻译源 provider 抽象与路由）
- [x] 免 Key 免费翻译源（内置 Google + 微软免 Key 公共端点，默认 microsoft，开箱即用，用户可切换）
- [x] 传统 API Key 配置（Google/微软填自有 Key，作为可选生效源）
- [x] 配置页翻译源选择 UI（源类型选择、生效源展示、连通性测试扩展）
- [x] LLM 响应风格扩展（openai/anthropic/ollama）、流式翻译、Popup 设置入口、译文 Markdown 渲染、LLM 类型统一（9 个功能事项全部 done，见 [releases/v0.2/](../releases/v0.2/index.md)）

> 本轮调整 v0.1「不内置任何模型接口」原则：内置传统翻译免 Key 公共端点作为用户可选的免费翻译源，无隐式自动回退。隐私声明相应更新。

## 变现接缝（2026-07-01 新增，2026-07-06 转向更新）

> 来源：[business/PRICING-PLAN.md](../business/PRICING-PLAN.md)。核心原则——**增长优先，前期纯免费；但路线图须为未来渐进式变现预留接缝。**

**变现接缝约束（v0.2-v0.5 功能设计须遵守）**：

- 历史记录、设置、Prompt 等数据结构须**可序列化、可增量同步**,不锁死纯本地,为未来接入云同步后端预留。
- 不在「翻译调用」上设计量/付费墙;翻译核心与免 Key 免费源保持 Free 全功能。
- 近期（增长优先阶段）**不设任何付费墙/软上限**，纯免费；软付费墙（Prompt 市场只读、术语表限额等）留作未来引入变现时再启用。

**WTP 驱动的路线图重排（2026-07-01 采纳，2026-07-06 推翻）**：

原按 WTP 把「历史 + 云同步 + 账号体系」提到 v0.3。战略转向后，v0.3 改为自动化插件市场发布流水线（增长优先），历史+云同步+账号体系延后至免费产品成熟、变现渐进引入时再建。WTP 排序保留作远期变现点参考（见 [business/PRICING-PLAN.md](../business/PRICING-PLAN.md)）。

**Pro 上线触发点**：不预设版本——免费产品成熟、WTP 访谈验证后渐进引入。Pro 上线前须同步更新隐私声明(账号、同步数据、支付通道)。
