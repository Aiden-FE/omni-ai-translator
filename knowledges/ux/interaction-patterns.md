# 交互模式 — LLM Translator

## 划词翻译（核心，v0.1）

### 触发模型：交互触发，不自动翻译

- 用户在网页选中文本 → `mouseup` 释放
- 选区右下方出现一个浮动小触发按钮（圆形，「译」字）
- **用户点击触发按钮**才开始翻译，避免打扰阅读流
- 选区为空或超 5000 字符时不出现触发按钮

### 流程

```
选中释放 → 出现触发按钮（pageX+8, pageY+8）
  → 用户点击触发按钮
  → 触发按钮消失，浮层出现并显示"翻译中…"
  → 后台调用 LLM
  → 成功：浮层显示译文
  → 失败：浮层显示 "❌ <错误信息>"
```

### 目标语言

- 默认浏览器首选语言（`navigator.language`，如 zh-CN → 简体中文）
- 用户可在设置页覆盖

### 关闭

- 点击触发按钮/浮层外任意位置 → 清除
- 新选区触发 → 旧触发按钮与浮层移除

### 状态反馈

| 状态 | 反馈 |
|------|------|
| 待触发 | 圆形「译」按钮 |
| 加载 | "翻译中…" |
| 成功 | 译文文本 |
| 失败 | "❌ " + 错误（如未配置提供方、网络错误、HTTP 错误码） |

## 设置页（options）

### 提供方管理

- 添加 / 删除 / 编辑提供方
- 启用：单选启用一个提供方为当前翻译后端
- 测试连通：发送 "hello→中文" 最小请求，显示译文或错误

### 默认目标语言

- 文本输入，自由填写（如「中文」「English」），传入 Prompt

## 弹窗（popup）

- 轻量入口，说明使用方式 + 「打开设置」按钮
- 第一版不承载复杂交互

## 待规划交互（后续版本）

- 划词翻译浮层：重译、复制译文、切换目标语言（v0.1 暂不暴露，先验证核心）
- 快捷键触发
- 小窗输入翻译窗口
