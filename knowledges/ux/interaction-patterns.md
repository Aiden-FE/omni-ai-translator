# 交互模式 — LLM Translator

## 划词翻译（核心，v0.1）

### 触发模型：交互触发，不自动翻译

- 用户在网页选中文本 → `mouseup` 释放
- 选区右下方出现一个浮动小触发按钮（圆形，「译」字）
- **用户点击触发按钮**才开始翻译，避免打扰阅读流
- 选区为空或超 5000 字符时不出现触发按钮

### 流程

```
选中释放 → 出现触发按钮（pageX+8, pageY+8）
  → 用户点击触发按钮
  → 触发按钮消失，浮层出现并显示"翻译中…"
  → 后台调用 LLM
  → 成功：浮层显示译文
  → 失败：浮层显示 "❌ <错误信息>"
```

### 目标语言

- 默认浏览器首选语言（`navigator.language`，如 zh-CN → 简体中文）
- 用户可在设置页覆盖

### 关闭

- 点击触发按钮/浮层外任意位置 → 清除
- 新选区触发 → 旧触发按钮与浮层移除

### 状态反馈

| 状态 | 反馈 |
|------|------|
| 待触发 | 圆形「译」按钮 |
| 加载 | "翻译中…" |
| 成功 | 译文文本 |
| 失败 | "❌ " + 错误（如未配置提供方、网络错误、HTTP 错误码） |

## 设置页（options）

### 翻译源管理（v0.2 升级）

- **当前生效源横幅**（顶部，`role="status"`，`data-state` 切换两态）：消费 `get-active-sources` / `set-active-source` 消息契约。
  - **兜底态**（生效源为 `builtin:*` 内置免 Key 源）：显示「免 Key 兜底」+ 隐私提示「未配置自有源，待翻译文本将外传到 Google / 微软完成翻译」+「配置自有源 →」锚点引导；附「测试连通」按钮测试当前内置源，结果 inline 展示。
  - **自有源态**：显示源名称 +「翻译请求将发送到该翻译源」。
- **源卡片**：添加 / 删除 / 编辑，每卡含名称、源类型下拉、baseUrl、apiKey。
  - 启用按钮切换生效源；已启用的自有源再次点击 → 回兜底（避免「启用后无法回到兜底」死路）；删除当前生效源时先回兜底再删。
- **4 类源类型下拉**：OpenAI 兼容（云端）/ Ollama（本地）/ Google 翻译 / 微软翻译。LLM 类（openai-compatible / ollama）显示「模型名」字段，传统类（google / microsoft）不显示；切换类型时 baseUrl 命中已知默认值则自动替换为新类型默认值。
- **连通性测试**：每卡「测试连通」复用 `test-provider` 通道，覆盖 LLM / 传统 API Key / 免 Key 兜底源，结果 inline 展示（✅ 译文 / ❌ 错误，按 `ok` / `err` 着色）。
- 顶部提示行：「未配置时使用免 Key 兜底翻译；配置自有源后覆盖兜底。」（取代 v0.1「不内置任何模型接口」提示）

### 默认目标语言

- 文本输入，自由填写（如「中文」「English」），传入 Prompt；留空时自动取浏览器首选语言。

## 弹窗（popup）

- 轻量入口，说明使用方式 + 「打开设置」按钮
- 第一版不承载复杂交互
