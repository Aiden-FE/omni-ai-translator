# PRD:正式产品名定稿（5-product-name）

> 版本:v0.3 第 5 项 · 优先级 P0 · 关联 Issue #53 · 创建日期 2026-07-07 · 状态 draft

## 1. 摘要

v0.3 商店首发前，需将内部代号 "LLM Translator" 正式定稿为可上架的产品名——以 "AI Translator" 为基底叠加区分词（命中翻译类热搜关键字、与其他产品区分），并同步 manifest name、popup/options UI 品牌字样、Chrome 商店名三处统一。本文档给出候选方向、定稿决策流程与定稿后需同步的点，不替代评审敲定最终名。

## 2. 联系人

| 角色 | 负责人 | 备注 |
|---|---|---|
| 产品 | Prodflow | 候选方向、定稿流程、验收 |
| 前端 | 待分配 | manifest name 同步、popup/options 品牌字样（与第 1 项 ui-rewrite 协同落地） |
| 发布 | 待分配 | Chrome 商店 listing 标题（与第 6 项 store-listing 协同） |
| 法务/检索 | 待分配 | 商标检索 + Chrome 商店重名检索 |

## 3. 背景

**上下文**：项目内部代号 "LLM Translator"（仓库 Aiden-FE/llm-translator），wxt.config.ts manifest.name 当前值为 `'LLM Translator'`，popup/options 顶部品牌字样也使用该名。v0.3 是增长优先阶段的商店首发版本，产品即将从"开发中"走向"商店可发现"。

**为什么现在做**：roadmap v0.3 明确要求"正式产品名确定：以 AI Translator 为基底叠加区分词，上架前定稿"。商店首发（第 3 项 release-pipeline、第 6 项 store-listing）依赖品牌名定稿才能上架；UI 美化（第 1 项 ui-rewrite）的品牌字样也需要定稿名才能统一呈现。名字不定，三处无法统一。

**变化点**：
- 当前名 "LLM Translator" 中 "LLM"（大语言模型）对普通阅读用户不友好——普通用户不熟悉这个术语，影响商店点击率与第一印象。
- Chrome 商店翻译类竞品密集，"AI Translator" 等通用名已被大量占用，需叠加区分词避免重名、提升可搜索性。
- 仓库名（Aiden-FE/llm-translator）保持不变，仅产品面向用户的品牌名变更。

## 4. 目标

**目标**：正式产品名定稿（含区分词），manifest name + UI 品牌字样 + Chrome 商店名三处统一，商标与商店重名检索无冲突。

**为什么重要**：品牌名是用户在商店看到的第一印象，也是搜索发现的关键入口。名字不友好或与竞品撞名会直接损失流量与信任；三处不统一会让用户困惑（商店装的是一个名、插件里看到另一个名）。战略明确"增长优先阶段产品美观可用、以发布速度抢占市场"——一个命中热搜且可区分的品牌名是商店分发增长的基础。

**战略对齐**：对齐策略文档"增长优先、前期纯免费聚焦产品美观可用、靠发布速度与免费产品力抢占市场"——品牌名定稿是商店首发（增长渠道）的前置条件。

**SMART OKR**：
- O：品牌名统一上架，用户在商店与插件内看到一致且可区分的品牌名
  - KR1：正式产品名定稿，含区分词，命中翻译类热搜关键字（检索确认可搜索性）
  - KR2：manifest name（wxt.config.ts）+ popup/options UI 品牌字样 + Chrome 商店名三处统一为定稿名
  - KR3：商标检索 + Chrome 商店重名检索均无冲突（检索记录留存）

## 5. 细分市场

按待完成任务定义用户：

- **商店发现用户（首次安装）**：任务是在 Chrome 商店搜索"翻译"类关键字、快速判断是否安装。约束：对 "LLM" 等技术术语不熟悉，靠名字第一印象与搜索排名决定是否点进。品牌名需命中热搜、一眼可懂。
- **已安装用户**：任务是在插件 popup/options 中确认"我装的是哪个翻译工具"。约束：品牌名需在 UI 中一致呈现，不与商店名产生混淆。
- **技术用户/开发者（开源社区）**：任务是找到仓库、理解产品定位。约束：仓库名保持 llm-translator 不变，品牌名与仓库名允许不同（README 说明对应关系即可）。

约束：Chrome 商店翻译类竞品密集，通用名易重名被拒；商标注册情况需检索；名字需中英文场景均可理解（商店名以英文为主）。

## 6. 价值主张

| 客户任务 | 现状痛点 | 升级后收益 |
|---|---|---|
| 商店搜索发现插件 | "LLM Translator" 不命中大众热搜词，普通用户不搜索 "LLM" | 名字含翻译类热搜关键字，搜索可发现性提升 |
| 一眼判断产品用途 | "LLM" 术语门槛高，普通用户不确定是什么 | "AI Translator"+ 区分词，一看即知是 AI 翻译工具 |
| 与竞品区分 | "AI Translator" 等通用名已被大量占用，难以区分 | 区分词让品牌在密集竞品中可辨认 |
| 安装后确认品牌一致 | 商店名与插件内品牌名可能不一致 | 三处统一，用户全程看到同一品牌名 |
| 商标与重名安全 | 未检索可能上架被拒或侵权 | 检索确认无冲突，上架无阻 |

**比竞品优在哪**：多数翻译插件用通用名（如 "Translate"、某引擎名），缺乏辨识度。本产品以 "AI Translator" 为基底叠加有辨识度的区分词，既命中 "AI 翻译" 热搜趋势，又在密集竞品中有独特性——配合 v0.3 美化后的视觉体验，形成"名字好找 + 界面好看"的双重第一印象优势。

## 7. 解决方案

### 7.1 UX / 原型（高层流程）

定稿决策流程（非 UI 交互流程，本事项为命名决策）：

```
候选方向梳理 → 商标检索 + Chrome 商店重名检索 → 评审敲定 → 三处同步
```

1. **候选方向梳理**：以 "AI Translator" 为基底，列出区分词候选方向（如命中翻译/语言/AI 等热搜修饰词，或体现产品差异化特征的修饰词），每条标注命中热搜词与可区分性评估。
2. **检索**：对每条候选执行商标检索（目标市场/类别）与 Chrome 商店重名检索，排除冲突项。
3. **评审敲定**：产品评审选定最终名（本 PRD 不替代评审决策，不硬编最终名）。
4. **三处同步**：定稿后同步 manifest name + UI 品牌字样 + 商店名（见 7.2）。

UI 品牌呈现的交互与视觉细节（popup/options 顶部品牌字样位置、字号、与新设计系统协同）由 prodflow-ux-evaluate 补全，品牌字样随第 1 项 ui-rewrite 落地（见「## UX 设计」章节）。

### 7.2 核心功能

1. **候选方向输出**：给出 "AI Translator" + 区分词的候选方向列表（区分词方向示例：翻译类热搜修饰词、语言类修饰词、产品差异化特征修饰词等），每条标注命中热搜词与可区分性，供评审决策。不在文档中敲定最终名。
2. **定稿后三处同步**：
   - **manifest name**：wxt.config.ts 的 `manifest.name` 字段同步为定稿名（当前值 `'LLM Translator'`）。
   - **UI 品牌字样**：popup 顶部、options 顶部品牌呈现同步为定稿名（品牌字样随第 1 项 ui-rewrite 重构落地，需协同）。
   - **Chrome 商店名**：商店 listing 标题同步为定稿名（与第 6 项 store-listing 协同，listing 素材排在其后）。
3. **检索记录留存**：商标检索结果与 Chrome 商店重名检索结果留存（作为定稿依据与上架合规佐证）。
4. **README 品牌名对应说明**：仓库 README 补充"仓库名 llm-translator / 正式产品名 XXX"对应说明（仓库名不变，避免社区困惑）。

### 7.3 技术

- **入口**：`wxt.config.ts` 的 `manifest.name` 字段（当前 `'LLM Translator'`，第 12 行）。
- **复用**：popup/options 品牌字样复用既有品牌呈现位置（popup 顶部、options 顶部），品牌字样随第 1 项 ui-rewrite 重构落地，不单独新增 UI 组件。
- **类型**：无类型变更（name 为字符串字面量）。
- **存储**：无存储变更（品牌名为编译期常量，不涉及 chrome.storage）。
- **兼容**：仓库名（Aiden-FE/llm-translator）保持不变；manifest.name 变更不影响已安装用户配置（chrome.storage 数据与 name 无关）；跨浏览器（Firefox/Edge）商店名各自同步（与第 7 项 cross-browser-build 协同）。

### 7.4 假设

- 假设 "AI Translator" + 区分词的方向能通过商标与商店重名检索（若全部冲突，需回评审调整候选方向）。
- 假设品牌字样随第 1 项 ui-rewrite 落地的时间窗口足够（不阻塞 manifest name 与商店名先同步）。
- 假设 Chrome 商店允许定稿名上架（重名检索通过即视为可上架）。
- 假设仓库名不变不影响商店审核（商店审核关注 manifest name 与 listing，不要求仓库名一致）。

## 8. 发布

**时间范围**：候选方向与检索约 0.5 周；定稿后三处同步随 ui-rewrite(1) 与 store-listing(6) 落地，不单独占完整周期。

**第一版（v0.3-5）包含**：
- 候选方向列表（"AI Translator" + 区分词，含热搜命中与可区分性评估）
- 商标检索 + Chrome 商店重名检索（记录留存）
- 评审敲定最终名
- manifest name 同步（wxt.config.ts）
- UI 品牌字样同步（随 ui-rewrite 落地）
- Chrome 商店名同步（随 store-listing 落地）
- README 品牌名对应说明

**未来版本**：跨浏览器（Firefox/Edge）商店名各自同步（随 cross-browser-build 落地）。
