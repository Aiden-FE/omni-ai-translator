# PRD:商店 Listing 素材（6-store-listing）

> 版本:v0.3 第 6 项 · 优先级 P1 · 关联 Issue #54 · 创建日期 2026-07-07 · 状态 draft

## 1. 摘要

本文档定义 LLM Translator 浏览器扩展上架 Chrome Web Store 所需的 Listing 素材清单与准备要求，涵盖商店图标、截图、描述文案、分类与关键词五类资产。增长优先阶段商店分发是核心增长渠道，Listing 质量直接决定用户能否在商店发现产品、以及看到后是否愿意安装。截图须在 UI 美化（第 1-2 项）完成后截取，以反映焕新后的明亮鲜艳视觉。

## 3. 背景

**上下文**：LLM Translator 处于增长优先阶段，战略明确「商店分发」是增长渠道之一，与口碑传播并列。v0.3 迭代主题为「商店首发与 UI 美化」——目标是打通「用户在商店发现产品 → 安装 → 使用」这条完整业务链。Chrome Web Store 上架需要一套 Listing 素材，这些素材是商店页面的门面，直接影响搜索发现率与安装转化率。

**为什么现在做**：v0.3 首次上架商店，此前项目从未准备过商店素材——当前 `wxt.config.ts` 的 manifest 甚至未声明 `icons` 字段，项目内也不存在任何图标文件。上架前必须补齐这些资产。同时，v0.3 同步进行应用全面美化（shadcn/ui + tailwindcss 重构、弃灰白黑冷色改明亮鲜艳主题），截图必须在美化完成后截取，否则商店展示的是旧视觉，无法体现「产品美观是增长抓手」的战略意图。

**变化点**：
- 项目从「仅 GitHub 开源」迈向「商店分发」渠道，Listing 素材是首次需要面向非技术普通用户的产品展示。
- 图标需多尺寸（16/32/48/128px）适配工具栏、扩展管理页、商店列表等场景。
- 描述文案需区分简短描述（商店列表一眼可见）与详细描述（点击展开后展示），面向普通用户说人话。
- 关键词须命中翻译类搜索热词，提升商店内被搜索到的概率。

## 4. 目标

**目标**：商店 Listing 素材齐备（图标/截图/描述/分类/关键词），支撑 Chrome Web Store 上架审核通过并最大化商店发现与转化。

**为什么重要**：增长优先阶段，商店是用户发现产品的主要入口。Listing 是用户在商店看到的第一印象——图标决定列表页点击率，截图决定详情页停留与安装意愿，关键词决定能否被搜到。素材缺失或质量差直接导致上架受阻或转化低下，浪费发布基建（第 3 项）打通的自动化上架能力。

**战略对齐**：对应战略文档「增长 → 商店分发」与「产品美观是增长抓手」——Listing 是商店分发的落地载体，截图须展示美化成果，使美观这一增长抓手在商店页面可见。

**关键结果（SMART OKR）**：
- O：Listing 就绪支撑商店发现与转化
  - KR1：图标（16/32/48/128px）、截图（≥1 张 1280x800 或 640x400）、简短描述（≤132 字符）、详细描述、分类、关键词全部齐备
  - KR2：截图反映美化后（第 1-2 项完成）的明亮鲜艳视觉，展示划词翻译与配置入口等核心场景
  - KR3：关键词覆盖翻译/AI/划词等翻译类搜索热词方向，提升商店内搜索命中

## 5. 细分市场

按待完成任务定义用户：

- **商店搜索者（普通阅读用户）**：任务是在 Chrome Web Store 搜索「翻译」或「translate」找到一款好用的翻译插件。他们在列表页扫图标和简短描述做快速判断，点进详情页看截图和详细描述决定是否安装。约束：注意力短，几秒内做判断；不懂技术术语（如 LLM、Ollama），需要看到「能翻译、好用、好看」。
- **商店浏览者（技术用户/开发者）**：任务是在商店或技术社区发现支持自带 Key、接本地模型的翻译插件。他们会看详细描述中的能力清单与隐私说明。约束：关注模型可控性与隐私，描述需提及 BYO Key 与本地模型支持。

约束：Chrome Web Store 对 Listing 素材有格式与尺寸要求（图标 PNG、截图 1280x800 或 640x400、简短描述 ≤132 字符）；素材须与实际产品功能一致，不可虚假宣传；产品名须以第 5 项定稿为准。

## 6. 价值主张

| 客户任务 | 现状痛点 | 升级后收益 |
|---|---|---|
| 在商店发现产品 | 项目无商店 Listing，无法被搜索到 | 关键词命中翻译类热词，商店内可被搜索发现 |
| 快速判断是否值得安装 | 无图标无截图，列表页无吸引力 | 商店图标醒目、截图展示美化后视觉，一眼可见产品力 |
| 了解产品能做什么 | 无描述文案，能力不可知 | 简短描述一句话讲清核心价值，详细描述列出关键能力 |
| 评估是否安全可信 | 无分类无信息，缺乏可信度 | 分类归属「生产力工具」，描述中说明隐私可控（Key 仅存本地） |

**比竞品优在哪**：多数翻译扩展的商店截图展示的是陈旧灰白界面，视觉同质化严重。LLM Translator 经 v0.3 美化后采用明亮鲜艳主题，截图在列表页中视觉辨识度更高——战略明确「在中小竞品密集的市场以视觉吸引力突围」，Listing 截图是这一突围的第一展示面。同时，BYO Key + 本地模型 + 免 Key 开箱即用的组合在竞品中少见，描述中突出这一差异化。

## 7. 解决方案

### 7.1 UX / 原型（高层流程）

商店 Listing 的「用户流程」发生在 Chrome Web Store 页面：用户搜索或浏览 → 列表页看到图标 + 简短描述 → 点击进入详情页 → 浏览截图 + 详细描述 + 分类/关键词 → 点击安装。

素材准备流程（内部）：
1. 确认第 1-2 项（ui-rewrite + visual-theme）已完成，应用视觉焕新就绪
2. 确认第 5 项（product-name）已定稿，用于商店标题
3. 设计并生成商店图标（128x128 主图 + 多尺寸）
4. 在美化后的应用中截取核心场景截图
5. 撰写简短描述与详细描述（方向稿，待评审定稿）
6. 确定分类（生产力工具）与关键词
7. 汇总素材提交商店开发者后台

详细视觉设计、截图构图与文案最终稿由 prodflow-ux-evaluate 补全（见「## UX 设计」章节）。

### 7.2 核心功能

1. **商店图标（多尺寸）**：
   - 需 16/32/48/128px 四尺寸 PNG 图标。其中 128x128 为商店主展示图标，16/32/48 用于扩展工具栏与管理页。
   - 当前项目无任何图标文件，`wxt.config.ts` manifest 也未声明 `icons` 字段——须新增图标资产并声明。
   - 图标视觉须与第 2 项（visual-theme）定稿的明亮鲜艳主题一致，体现产品调性。
   - 方向：以「译」字或翻译符号为视觉核心，配色呼应主题主色/强调色（具体设计稿待评审）。

2. **截图**：
   - 尺寸 1280x800（推荐）或 640x400，至少 1 张，建议 3-5 张覆盖核心场景。
   - 须在美化完成后截取，反映明亮鲜艳新视觉。
   - 场景方向（待评审定稿）：划词翻译浮层展示译文、popup 设置面板（翻译源配置）、options 配置页、连通性测试成功态等。
   - 截图为静态 PNG/JPG，提交商店开发者后台，不打包进扩展。

3. **描述文案**：
   - 简短描述：≤132 字符，一句话讲清产品是什么。方向——「支持 AI 大模型与免费翻译源的浏览器划词翻译插件，开箱即用」（具体文案待评审）。
   - 详细描述：展开后展示，列出核心能力清单。方向涵盖：划词翻译、免 Key 开箱即用（默认微软）、自带 Key 接云端/本地大模型、隐私可控（Key 仅存本地）、Prompt 自定义、多翻译源切换（具体文案待评审）。
   - 文案面向普通用户说人话，避免纯技术术语，但详细描述中可提及 BYO Key / 本地模型以吸引技术用户。

4. **分类**：
   - 选「生产力工具」（Productivity）。翻译插件属生产力工具范畴，该分类用户搜索量高。

5. **关键词**：
   - 围绕翻译/AI/划词方向。方向词（待评审定稿）：翻译、translate、AI 翻译、划词翻译、浏览器翻译、大模型翻译、免费翻译。
   - 目标是命中用户在 Chrome Web Store 搜索翻译类扩展时的常见用词。

### 7.3 技术

- **入口**：图标文件存放 `public/` 目录（WXT 约定），在 `wxt.config.ts` 的 `manifest.icons` 字段声明各尺寸路径。当前 manifest 无 `icons` 声明，须新增。
- **复用**：16/32/48px 图标同时用于扩展运行时（工具栏/管理页）与商店 Listing，不重复制作；128px 专为商店主图。截图与描述/关键词/分类为商店后台填写项，不进入代码仓库（可选归档到 `releases/v0.3/6-store-listing/` 供记录）。
- **类型**：图标 PNG 格式（商店要求）；截图 PNG 或 JPG。不涉及 TS 类型变更。
- **存储**：图标作为静态资产由 WXT 打包进扩展；截图与文案为商店开发者后台提交项，不存入 chrome.storage。
- **兼容**：图标声明同时满足 Chrome 商店要求与扩展运行时需要；跨浏览器（Firefox/Edge）图标要求一致，第 7 项（cross-browser-build）可直接复用。

### 7.4 假设

- 假设第 1-2 项（ui-rewrite + visual-theme）在本项截图前完成，截图能反映最终美化视觉。若美化延期，截图相应后移。
- 假设第 5 项（product-name）在本项商店标题填写前定稿。若产品名未定，商店标题暂用占位，定稿后更新。
- 假设 Chrome Web Store 简短描述字符上限为 132 字符（商店现行规则），若规则调整以商店实际为准。
- 假设截图场景选取的 popup/options 页在美化后视觉达标，无需额外修饰。
- 假设图标设计可复用第 2 项（visual-theme）产出的设计 token（主色/强调色），保持视觉统一。

## 8. 发布

**时间范围**：约 0.5-1 周（素材准备为主，依赖第 1-2 项美化完成与第 5 项产品名定稿）。

**第一版（v0.3-6）包含**：
- 商店图标 4 尺寸（16/32/48/128px PNG）+ manifest `icons` 声明

**未来版本（v0.4+）**：
