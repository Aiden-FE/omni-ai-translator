## 1. 摘要

本 PRD 定义「免 Key 免费翻译源」：内置 Google 与微软翻译的免 Key 公共端点作为 provider，作为用户可选的免费翻译源，使用户即使没有 LLM API Key 或本地模型也能开箱即用、直接划词翻译。这是 v0.1「不内置任何模型接口」原则在 v0.2 的边界调整——为降低首次使用门槛，内置传统翻译的公共端点作为可选免费源。

## 3. 背景

v0.1 明确「不内置任何模型接口」，用户必须自行配置 OpenAI 兼容接口或本地 Ollama 才能翻译。这对普通阅读用户门槛过高——许多人没有 API Key，也不知道如何本地部署模型，导致插件装上即不可用。

v0.2 决定调整这条原则：**内置传统翻译接口的免 Key 公共端点作为用户可选的免费源**，让用户开箱即用。选择传统翻译接口（而非内置某个 LLM）作为免费源的原因：

- 传统翻译接口（Google / 微软）有长期稳定的非官方公共端点，无需 Key 即可调用，适合作为免费可选项。
- LLM 接口几乎都需要 Key，无法做到真正免 Key 开箱即用。
- 免费源只求「能用」，传统机翻质量足够应急；追求质量时用户仍可配置 LLM。

本 PRD 依赖功能事项 1（unified-adapter）提供的 `TranslationProvider` 接口与注册表，在其上实现 google / microsoft 两个传统 provider 并注册为可选免费源。

**关键现实与风险**：所谓「免 Key 公共端点」多为**非官方端点**（如 Google 翻译网页端点、Edge 浏览器翻译端点），稳定性有限：可能被限流、封禁、或在部分地区不可达（Google 翻译在中国大陆不可达）。本 PRD 以「多源可选 + 明确错误提示 + 人工切换」应对，**不做自动降级或隐式回退**。

## 4. 目标

**目标**：用户即使没有 LLM API Key，也能通过选择内置的免费翻译源（google / microsoft）开箱即用、直接划词翻译。

**对齐愿景**：产品策略定位「让任何人都能用上最好的大模型翻译」——但「任何人」的前提是先能「用上」。免费源降低了零门槛体验，让普通用户先用起来，再引导他们配置 LLM 获得更好质量。

**关键结果（SMART）**：

| 编号 | 关键结果 | 衡量方式 |
|---|---|---|
| KR1 | 至少 2 个免 Key 传统 provider（google、microsoft）实现并注册到适配层，作为用户可选源 | 代码审查 |
| KR2 | 全新安装默认选中 microsoft，划词翻译返回译文，无需任何配置 | e2e：全新状态划词成功 |
| KR3 | 免费源失败时返回明确错误提示，引导用户切换或配置其它源（不自动回退） | 单元测试覆盖错误路径 |
| KR4 | 隐私声明已更新，明示选择免费源后文本外传到 Google/微软 | 隐私声明审查 |

## 5. 细分市场

- **普通阅读用户**（待完成任务：阅读外文页面，没有 API Key，只想马上看懂）：本 PRD 的核心受益者。零配置即可翻译（默认 microsoft），消除首次使用门槛。
- **技术用户 / 开发者**：免费源作为「应急方案」，他们仍可配置自有 LLM/Key 追求质量与隐私。本 PRD 不影响其既有路径。

**约束**：
- 免费源为非官方公共端点，稳定性与可达性无保证。
- 中国大陆环境下 Google 翻译不可达，需 microsoft（或其它可达源）作为默认/备选。
- 必须在隐私层面如实告知：选择免费源后，待翻译文本会外传给 Google/微软。

## 6. 价值主张

| 客户任务 / 需求 | 客户获得 | 客户避免的痛点 | 我们做得更好的地方 |
|---|---|---|---|
| 装上插件就想用 | 零配置开箱即用（默认 microsoft），划词即译 | 被迫先研究 API Key / 本地模型 | 多数翻译插件需配 Key；免费源让首次体验顺畅 |
| 应急看懂外文 | 传统机翻质量足够应急 | 无可用源时空白 | 内置多源可选，单点不可达可人工切换 |
| 知道数据去哪 | 隐私声明明示外传到哪 | 不知文本被发给谁 | 如实告知并提供配置自有源以保护隐私的路径 |

## 7. 解决方案

### 7.1 UX / 原型

用户侧无新增独立界面。免费源体现为「配置页可选源之一」：用户可在配置页（功能事项 4）的源选择列表中选择 google / microsoft 免费源。全新安装默认选中 microsoft，划词翻译正常展示译文。配置页会展示当前生效源（含免费源）。

错误反馈：当免费源不可达时，划词浮层展示明确提示（如「免费翻译源不可达（可能受限流或地域限制），请在配置页切换其它源或添加自有翻译源」），由统一错误模型（功能事项 1）产出。**不自动回退到其它源**。

> 详细 UX 设计（差异性交互、状态、反馈、可访问性、视觉原型）待 issue 创建后由 prodflow-prd revise 模式补全（参考 prodflow-ux-evaluate 技能产出）。

### 7.2 核心功能

**7.2.1 免 Key 传统 Provider 实现**

在功能事项 1 的 `TranslationProvider` 接口上实现两个 provider：

- **google**：调用 Google 翻译免 Key 公共端点（非官方网页端点），解析返回结构提取译文。
- **microsoft**：调用微软翻译免 Key 端点（如 Edge 浏览器翻译端点等可达方案），解析提取译文。

两个 provider 均**无需用户填写 Key**，端点内置。实现细节（确切 URL、请求参数、响应解析）由研发在实现时确定并验证可达性，本 PRD 只约束行为契约：输入 `TranslateRequest`，输出 `TranslateResult`，失败归入统一错误模型。

**7.2.2 默认生效源与切换逻辑**

- 全新安装、用户未做任何选择时，**默认选中 microsoft**（显式默认值，保证开箱即用）。
- google、microsoft 作为可选源与 LLM 源并列出现在配置页源选择列表中，用户可随时主动切换。
- **无隐式自动回退**：当前生效源失败时，不自动切换到其它源，仅返回错误提示；切换由用户人工操作。
- 用户配置并启用任意自有源（LLM 或传统 API Key）后，生效源为用户选择项；免费源仍在可选列表中，用户可随时切回。

**7.2.3 错误处理与提示**

免费源失败时按统一错误模型归类，**不自动回退**：

- 网络错误 / 源不可达（含地域不可达）：提示「免费翻译源不可达，请在配置页切换其它源或添加自有翻译源」。
- 限流：提示「免费源繁忙，请稍后再试或配置自有源」。

**7.2.4 host_permissions 声明**

免费源域名需在 manifest `host_permissions` 声明，确保 background 能跨域请求。由研发在 wxt.config.ts / manifest 配置中落实。

### 7.3 技术

- Provider 实现位于 `shared/`（与 LLM provider 同层），由 background 经适配层调用。
- 端点内置为常量，不暴露给用户编辑（免费源不可编辑，只能选择/切换或被自有源替换为生效项）。
- 响应解析需处理非官方端点的非标准 JSONP / 数组结构，做好容错。
- 生效源切换契约：在 background 暴露消息接口供配置页（功能事项 4）调用——`getActiveSources`（返回含免费源的可用列表及当前生效源）、`setActiveSource`（切换生效源）。

### 7.4 假设

| 假设 | 验证方式 | 失败信号 |
|---|---|---|
| Google / 微软存在可用的免 Key 公共端点 | 研发实现时实测 | 两源均不可用，免费源不可行 |
| microsoft 端点在中国大陆可达 | 实测 | 大陆用户默认 microsoft 不可用，需引导切换 |
| 非官方端点响应结构可稳定解析 | 实测 + 容错 | 响应结构频繁变动致解析失败 |
| 用户接受选择免费源后文本外传到 Google/微软 | 隐私声明明示 | 用户反馈隐私顾虑 |

## UX 设计

### UX 依据

- Issue: `#2`（https://github.com/Aiden-FE/llm-translator/issues/2）
- 版本: v0.2
- 知识来源: [knowledges/ux/index.md](../../../knowledges/ux/index.md)（设计原则：轻量不打扰、隐私透明）、[knowledges/ux/interaction-patterns.md](../../../knowledges/ux/interaction-patterns.md)（划词浮层）、[knowledges/ux/accessibility.md](../../../knowledges/ux/accessibility.md)
- 设计假设: 用户接受「全新安装默认选中 microsoft 免费源（文本外传到微软）」的默认行为，只要在配置页如实告知并可切换即可——该假设待用户验证，若用户反馈默认外传不可接受，需改为「首次使用时显式确认」

### 视觉设计

> 本 PRD 无新增独立用户界面（免费源为配置页可选源之一，配置页源选择 UI 由 [../4-source-picker-ui/PRD.md](../4-source-picker-ui/PRD.md) 承担）。沿用现有划词浮层视觉，不单独产出视觉原型。

### 交互逻辑（差异）

通用划词流程引用 [knowledges/ux/interaction-patterns.md](../../../knowledges/ux/interaction-patterns.md)，不重复展开。本 PRD 差异：

1. **默认无感启用**：全新安装默认选中 microsoft 免费源，划词翻译正常返回译文，**浮层不额外标注「免费源」字样**，遵循「轻量不打扰」原则——译文正常展示，体验与配置源一致。
2. **隐私透明落在配置页**：免费源的「文本外传到 Google/微软」告知放在配置页（当前生效源展示处），不在每次翻译的浮层中重复打扰。配置页提示文案约束见 [../4-source-picker-ui/PRD.md](../4-source-picker-ui/PRD.md)。
3. **免费源失败反馈**：沿用 [../1-unified-adapter/PRD.md](../1-unified-adapter/PRD.md) 统一错误模型，免费源失败按「源不可达 / 限流」反馈，引导文案强化「请在配置页切换其它源或添加自有翻译源」，**不自动回退**。

### 业务约束

- 免费源对用户不可编辑（端点内置），可在配置页选择/切换或被自有源替换为生效项。
- **无隐式自动回退**：生效源失败仅提示，切换由用户人工操作。
- 默认生效源：microsoft（全新安装）。
- 隐私约束：选择免费源期间，待翻译文本外传给 Google/微软，配置页必须如实反映当前生效源及外传去向。

### 状态变化（差异）

浮层状态沿用第 1 项四类错误子状态。免费源场景下高频出现的两类：

| 状态 | 反馈 |
|---|---|
| 成功（免费源） | 译文文本（与配置源成功态无差异） |
| 失败-源不可达（免费源） | ❌ 翻译源不可达 + 引导：请在配置页切换其它源或添加自有翻译源 |
| 失败-限流（免费源） | ❌ 翻译源繁忙 + 引导：请稍后再试或配置自有源 |

### 设备适配（差异）

无差异。浮层响应式沿用 [knowledges/ux/design-system.md](../../../knowledges/ux/design-system.md)。

### 可访问性（差异）

- 免费源状态对辅助技术无额外要求（浮层不标注免费源）。
- 隐私告知在配置页以文字呈现，不依赖颜色，需满足对比度 AA（约束第 4 项配置页实现）。

### 验收补充

- 全新安装默认选中 microsoft，划词翻译返回译文，浮层无「免费源」额外标注（轻量）。
- 免费源失败时返回「源不可达/限流」可读提示并引导切换或配置自有源（不自动回退）。
- 配置页如实展示免费源为当前生效源及外传去向（约束第 4 项）。
- 隐私声明已更新选择免费源后的外传行为（见 [../../../knowledges/product-wiki/privacy/PRIVACY-POLICY.md](../../../knowledges/product-wiki/privacy/PRIVACY-POLICY.md)）。
