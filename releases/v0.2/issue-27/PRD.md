# PRD — Issue #27 配置自有源点击无反应

| 项 | 内容 |
|---|---|
| 文档类型 | 需求文档（PRD） |
| Issue | #27 【AI】【bug】配置页 - 默认配置下「配置自有源」点击无反应 |
| 类型 | BUG；严重程度：一般 |
| 关联 PRD | #4 (source-picker-ui) |
| 迭代版本 | v0.2 |

## 1. 问题描述

默认配置（fresh install，providers 为空，生效源 builtin:microsoft 兜底，isFallback=true）下，配置页顶部生效源提示行的「配置自有源 →」是纯锚点 `<a href="#source-section">`。由于源区块 `#source-section` 默认已在视口内，点击后无可见反应——首屏入口失效，影响新用户上手。

复现步骤：全新安装/重置扩展至默认配置 → 打开扩展选项页 → 顶部生效源提示行点击「配置自有源 →」→ 无反应。

## 2. 期望行为

点击「配置自有源 →」后真正发起自有源配置：自动新建提供方卡片并聚焦名称输入（Issue 首选方案），而非无反应的锚点跳转。

## 3. 验收标准

| 编号 | 验收项 | 验收条件 | 优先级 |
|---|---|---|---|
| AC1 | 点击有反应 | 默认配置（isFallback=true）下点击「配置自有源 →」不再是无反应锚点跳转，而是触发新建 provider 卡片 | P0 |
| AC2 | 新建并聚焦 | 点击后自动新建一个提供方卡片，并聚焦该卡片名称输入框 | P0 |
| AC3 | 滚动入视口 | 新建卡片若在视口外，聚焦时自动滚动入视口 | P0 |
| AC4 | 横幅状态不变 | addProvider 后 activeSourceId 仍是 builtin（新卡片未激活），isFallback 保持 true，横幅正常展示兜底态 | P0 |
| AC5 | 聚焦可见 | 聚焦的名称输入框有可见 focus 样式（现有 CSS outline 已满足） | P1 |
| AC6 | 向后兼容 | 非兜底态（isFallback=false）下不显示「配置自有源」入口，行为不变 | P0 |
